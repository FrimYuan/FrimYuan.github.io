一种不错的屏幕适配方案——SmallestWidth适配

发表于 2018-08-21 更新于 2020-02-26 分类于 Android
本文字数： 4.4k 阅读时长 ≈ 4 分钟
屏幕适配方案

前言

在日常开发中我们免不了对App做屏幕适配，这篇文章简单总结一下屏幕适配。

基本概念

px （Pixel像素）：也称为图像元素，是作为图像构成的基本单元，单个像素的大小并不固定，跟随屏幕大小和像素数量的关系变化（屏幕越大，像素越低，单个像素越大，反之亦然）。所以在使用像素作为设计单位时，在不同的设备上可能会有缩放或拉伸的情况。

resolution（分辨率）：是指屏幕的垂直和水平方向的像素数量，如果分辨率是 1920x1080 ，那就是垂直方向有 1920 个像素，水平方向有 1080 个像素。

dpi（像素密度）：每英寸有多少个像素点，分为物理dpi（可以通过下面的公式计算得出）和系统dpi，这两个值不一定相同，系统dpi是由手机厂商设置的，并且可以在手机中修改。

density（密度）：Android中表示的是显示的逻辑密度

dip / dp（设备独立像素）：也可以叫做dp，长度单位，同一个单位在不同的设备上有不同的显示效果，具体效果根据设备的密度有关，详细的公式请看下面 。

sp：是Android推荐使用的字体大小单位，换算和dip相同，但是这里有个问题，sp受系统字体大小设置的影响，dip不会。什么意思呢，如果你在xml中设置TextView的textSize是20sp，显示的和效果图差不多，这时如果在手机系统设置应用中调整了字体大小，回到应用就会发现文字的大小就不是刚才设置的20sp了，会根据你在设置中的调整相应的放大或是缩小，所以如果你不想字体随着系统字体大小变化的话，那么设置字号时单位你也可以选择dip。

换算公式

dpi=√(分辨率宽²+分辨率高²)/屏幕尺寸（例：分辨率为320x240，2.5英寸的屏幕，Dpi=√(320²+240²)/2.5=160）

density = dpi / 160

px = dp x density

Android 系统默认定义的 dpi

ldpi mdpi hdpi xhdpi xxhdpi
分辨率 240x320 320x480 480x800 720x1280 1080x1920
dpi 120 160 240 320 480
density 0.75 1 1.5 2 3

SmallestWidth适配

好了，基本概念和计算公式都介绍完了，接下来该讲如何适配了，这里我用的是SmallestWidth适配（sw），指的是Android会识别屏幕分辨率的最小尺寸的dp值，然后根据结果去资源文件中找到并使用对应限定符文件夹下的资源文件。主要思路一句话总结就是等比缩放分辨率后再根据换算将px换成dp。

这个sw的计算公式为：sw = 分辨率最小尺寸 / density

例如，我这个模拟器是1080x1920的分辨率，dpi是480，那么我们通过density的计算公式得出：density = 480 / 160 = 3，接下来我们再计算sw：sw = 1080 / 3 = 360dp，接下来我们在项目中创建限定符sw为360的文件夹

不过我们并不用担心，表中加粗的是我遇到过的情况，你也可以根据实际情况自己计算，另外如果你的sw文件夹并没有涵盖所有情况也没有关系，系统会自动寻找与它相近的dimens使用。另外这里也体现出出sw适配的好处，虽然手机有很多分辨率，但是这种适配根本不需要考虑，只需要把通用的sw计算出来即可。

其实这块还有更简便的计算方法，例如上面的例子中设备的sw是360dp，表示手机最小宽度的dp值是360dp，接下在尺寸图给出的分辨率是1080x1920，我们就可以将360dp等分成1080份，表示为每个像素占360dp/1080px≈0.333dp，那么这时1080x1920的尺寸图中宽度为100px的控件在360dp下应为：100x0.333=33.3dip
